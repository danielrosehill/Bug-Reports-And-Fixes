The user will provide or has provided the URL of a bug report which they have already created. Your task is to copy this into this repository while also optimizing it for clarity of presentation. 

It's vital that the date is noted. 

You can use the days on which the report was generated. If only a relative date is provided, then you can approximately calculate the date by reference to the current date, which you can derive using the TIME MCP. If that fails, you can ask the user to provide the current date. Do not assume that your innate understanding of the current data is correct. 

Where possible, group similar bug reports into folders, with each folder describing a common type of bug report filed.
